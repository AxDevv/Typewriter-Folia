# Typewriter - Instructions Claude

## Objectif Principal
Rendre le plugin 100% compatible FOLIA sans oublier une seule classe.

## Méthodologie

### BMAD (Benchmark, Analyze, Modify, Verify)
Appliquer cette méthode pour toute analyse, code-review et vérification de performance:
1. **Benchmark** - Établir l'état actuel et les exigences
2. **Analyze** - Analyser en profondeur le code et les dépendances
3. **Modify** - Modifier uniquement ce qui est nécessaire pour FOLIA
4. **Verify** - Vérifier que les modifications fonctionnent

## Règles de Développement

- Modifier le minimum de code existant
- Vérifier systématiquement chaque fichier .kt
- Toujours lire et comprendre avant de modifier
- Utiliser des agents pour les tâches complexes

## Compatibilité FOLIA

Priorité absolue: toutes les classes doivent être compatibles FOLIA.
